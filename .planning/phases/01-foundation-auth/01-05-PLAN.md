Key contracts being added/finalized here:

`src/actions/members.ts` exports `joinTrip(token: string): Promise<{ tripId: string | null; error: string | null }>`. Logic:
1. await createClient() (server factory).
2. const { data: { user } } = await supabase.auth.getUser().
3. If !user, call supabase.auth.signInAnonymously(). If that fails, return { tripId: null, error: es.errors.genericNetwork }.
4. Re-read the user (after anonymous sign-in).
5. Look up trip by token: supabase.from('trips').select('id').eq('invite_token', token).single(). If no row, return { tripId: null, error: es.errors.invalidJoinToken }.
6. Upsert trip_members: supabase.from('trip_members').upsert({ trip_id: trip.id, user_id: user.id, role: 'member' }, { onConflict: 'trip_id,user_id' }). Return { tripId: trip.id, error: null }.

`src/components/auth/AnonymousUpgradeSheet.tsx` exports `<AnonymousUpgradeSheet open onOpenChange />`. Internally uses createClient() (BROWSER factory from Plan 03 — anonymous upgrade is documented as a client-side call per RESEARCH Pattern 4). Submits via React Hook Form + Zod.

`src/components/common/AnonymousBanner.tsx` exports `<AnonymousBanner />`. Wires AnonymousUpgradeSheet open state via local React state. Uses Zustand `useBannerStore` for session-scoped dismiss.

`src/stores/banner.ts` exports `useBannerStore` with `{ dismissed: boolean; dismiss(): void; reset(): void }`. NOT persisted (session-scoped per D-12: "Dismissal is session-scoped (re-appears next launch)").

`src/i18n/es.ts` extended with `es.anon.*` namespace from RESEARCH Pattern 9: pill, bannerHeading, bannerCta, upgradeSheetHeading, upgradeSheetBody, upgradeEmailLabel, upgradeSubmitCta, upgradeSuccessToast(email).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> /join/[token] | Untrusted token from URL; validated as UUID + RLS confirms membership eligibility. |
| Browser -> joinTrip server action | Untrusted token; joinTrip uses parameterized .eq query (no SQL injection surface). |
| Browser -> supabase.auth.updateUser | Untrusted email; Zod validates format before send. Supabase verifies email ownership via confirmation link. |
| trip_members row insert | RLS WITH CHECK enforces user_id = (SELECT auth.uid()) — user cannot insert membership for someone else. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Attacker uses joinTrip with a token they guessed | mitigate | Token is a UUIDv4 (122 bits entropy). Guessing one in <10^36 attempts is computationally infeasible. RESEARCH confirms invite_token is gen_random_uuid(). |
| T-05-02 | Elevation of Privilege | Anonymous user inserts a trip_members row for a trip they don't have the token for | mitigate | trip_members RLS INSERT policy: WITH CHECK (user_id = (SELECT auth.uid())) — user can only insert their own membership. The token-validity check is enforced by joinTrip looking up the trip by token first. |
| T-05-03 | Spoofing | Attacker registers an email that is already someone else's account during anonymous upgrade | mitigate | Supabase's updateUser({ email }) sends a confirmation email to the new address. If the email is already in use, Supabase returns an error (the action returns { error: ... } and the toast displays it). Phase 5 may add explicit "email already in use" copy in es.errors. |
| T-05-04 | Information Disclosure | Anonymous session cookie shared between two iPhones | accept | Anonymous sessions are tied to a single browser's cookie. If a user manually copies the cookie to another device, both devices share the same anonymous user.id — they see the same trip. This is the design (friend-circle product). Permanent accounts via magic link mitigate this in the long term. |
| T-05-05 | Repudiation | An anonymous user joins, vandalizes a trip, then disappears (no recovery contact) | accept | Trip creator can remove members via the trip_members DELETE policy (admin role). Phase 5 may add audit logging. |
| T-05-06 | Denial of Service | Bot floods joinTrip with valid tokens to create many anonymous users | accept (low-risk) | Supabase rate-limits signInAnonymously at the project level (default ~30/hour per IP). Phase 5 may add Vercel edge rate-limiting if abuse appears. Personal-circle product — low value target. |
| T-05-07 | Information Disclosure | The `error` query param in the redirect from /join/[token] -> / contains user-facing Spanish text → could be used to inject content via reflected XSS | mitigate | The welcome page maps `error` value to a fixed es.errors.* key — it does NOT render the param value directly. Even if an attacker crafts a URL like /?error=<script>, the welcome page only renders one of a fixed set of i18n strings (no echo of unknown values). |
| T-05-SC | Tampering | Zustand added previously in Plan 01; no new supply-chain risk here | n/a | Zustand was already audited in Plan 01 Package Legitimacy Audit (pmndrs/zustand, official org). No new dependency. |

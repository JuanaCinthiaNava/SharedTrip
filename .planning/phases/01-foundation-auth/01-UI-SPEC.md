## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tab-bar icon-to-label gap, pill horizontal padding |
| sm | 8px | Input internal padding, avatar-to-name gap, badge padding |
| md | 16px | Default element spacing, card body padding, form field gap |
| lg | 24px | Section padding, header top/bottom padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Full-bleed screen vertical padding top |
| 3xl | 64px | Bottom clearance above tab bar (content must not be obscured) |

Exceptions:
- Touch targets for tab bar items: minimum 44px height (iOS HIG compliance) — this exceeds the xl token but is a separate constraint.
- Bottom tab bar height: 56px fixed (contains 44px touch target + 12px visual padding).
- "Sin cuenta" pill height: 24px (sm vertical padding inside a rounded-full badge).

## Typography

Single typeface: **Inter variable**, loaded via `next/font/google`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 700 (bold) | 1.2 | Screen headings (welcome screen title, empty-state heading) |
| Heading | 20px | 600 (semibold) | 1.3 | Section titles, modal/sheet titles, card titles |
| Body | 16px | 400 (regular) | 1.5 | Primary readable text — descriptions, form help text, banner copy |
| Label | 14px | 500 (medium) | 1.4 | Input labels, tab bar labels, badge text, secondary metadata |

**Constraints:**
- Maximum 4 sizes declared above. No additional sizes may be introduced in Phase 1.
- Maximum 3 weights in use: 400 / 500 / 600–700. Weight 700 is Display only.
- Brand wordmark "SharedTrip": Inter italic, weight 600, `--color-primary` (coral). Rendered as text, not SVG.
- Tab bar labels: 12px is acceptable as a sub-token ONLY for the bottom tab bar icon labels (iOS convention) — this is the single exception to the 14px label floor.

## Color

**Theme: Tropical Sunset, dark-mode default. No light-mode toggle in Phase 1.**

All colors defined as Tailwind v4 `@theme` tokens in `globals.css`. No hex literals in component files — reference tokens only.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-bg` | `#0F1729` | Page background — every screen's root background |
| Secondary (30%) | `--color-surface` | `#1A2238` | Cards, sheets, bottom sheet backgrounds, tab bar background, input background |
| Accent primary (10%) | `--color-primary` | `#FF6B6B` | See reserved list below |
| Accent secondary | `--color-accent` | `#3DCCC7` | See reserved list below |
| Warning / notice | `--color-secondary` | `#FFB627` | "Sin cuenta" pill background, nudge banner background highlight |
| Foreground primary | `--color-fg` | `#FAFAFA` | All primary text |
| Foreground muted | `--color-fg-muted` | `#94A3B8` | Secondary text, inactive tab labels, placeholder text, footer copy |
| Destructive | `--color-destructive` | `#EF4444` | Sign-out destructive confirmation only |

**Accent primary (`--color-primary` coral) reserved for:**
1. Primary CTA buttons ("Entrar", "Continuar", "Enviar enlace")
2. Magic link email CTA button background
3. Active tab bar indicator (underline + icon tint)
4. Brand wordmark "SharedTrip" text color
5. PWA icon background (coral square with white "S")

**Accent secondary (`--color-accent` teal) reserved for:**
1. Inline text links
2. Success toast background
3. "Perfil" tab active state (alternate to coral — only if visual conflict with coral active tab arises; default is coral for all active tabs)

**Warning (`--color-secondary` mango) reserved for:**
1. "Sin cuenta" pill badge (anonymous session indicator)
2. Persistent upgrade nudge banner left border/accent stripe
3. Toast notifications for non-error warnings

**Nothing else** may use accent colors. Default interactive states use `--color-fg` on `--color-surface`.

## Interaction Contracts

### Bottom Tab Bar
- Active tab: coral underline 2px + coral icon tint + coral label text
- Inactive tab: `--color-fg-muted` icon + label
- Tap: immediate visual response (no delay); tab content swaps without page navigation animation
- Height: 56px fixed; positioned `fixed bottom-0` with `padding-bottom: env(safe-area-inset-bottom)` for iPhone home indicator
- Tab order (left to right): Docs / Itin / Gente / Perfil
- Tab labels: 12px Inter medium; icons: 24px Lucide

### Top Header
- Height: 56px; `sticky top-0`; background `--color-surface`; bottom border 1px `--color-fg-muted` at 20% opacity
- Left: trip name as tappable dropdown trigger (chevron icon right of text); text truncated at 160px max-width
- Right: user avatar (32px circle) — if anonymous, "Sin cuenta" pill (mango) appears left of avatar
- No back button in Phase 1 (single-level navigation within trip shell)

### Magic Link Form
- Email input: `type="email"`, autocomplete `email`, `inputmode="email"` for mobile keyboard
- Submit button: full-width, coral, disabled while loading; shows spinner (Lucide `Loader2` with `animate-spin`)
- Validation: Zod `z.string().email()` — inline error below input in `--color-destructive` at 14px
- Error display: shown on blur AND on submit attempt; not on keystroke
- After successful submit: form replaced by confirmation message (no navigation)

### "Sin cuenta" Pill
- Display condition: `session.user.is_anonymous === true` AND user is on any screen with the top header
- Hidden: on auth/welcome screen, on `/auth/check-email`
- Tap: opens `AnonymousUpgradeSheet`

### Upgrade Nudge Banner
- Display condition: same as "Sin cuenta" pill (anonymous session)
- Position: immediately below `TopHeader`, above tab content
- Dismiss: X button (session-scoped — reappears next launch)
- Animation: slides down from header on mount (CSS transition, 200ms ease-out)
- Hidden: on auth/welcome screen

### Toast Notifications
- Position: top-center on mobile (Sonner default)
- Duration: success = 4s; error = 6s; warning = 5s
- No stacking more than 2 toasts simultaneously

### Loading States
- Data-fetching regions: `Skeleton` component, matching the shape of the content that will appear
- Button loading: replace label with `Loader2` spinner (Lucide, `animate-spin`) — do not disable without showing spinner
- Full-screen loading (route transitions): none in Phase 1 (placeholder tabs load instantly)

### AlertDialog (sign-out)
- Destructive confirm button uses `--color-destructive` background
- Cancel is always the left / less prominent button
- Dialog triggered only once (no "Are you sure you're sure")

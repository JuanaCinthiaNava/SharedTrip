<objective>
Ship the typed-invite-code entry slice end-to-end and remove the magic-link path from v1. After this plan, a friend who has never used the app opens `/`, types a short code their group shared (e.g. `TEST-AB12`), and lands inside the trip with an anonymous session — zero email, zero login. The welcome screen, the join flow, and the data layer (Plan 01-08) now all speak `invite_code`.

Purpose: Deliver the re-scoped Phase 1 entry path (entry-model-invite-code note, 2026-06-01) on top of the shipped [[anon-join-architecture]]. Only the trip-resolution input changes (uuid token -> typed code); the anonymous sign-in + service-role membership insert (the 4 coordinated fixes) stay byte-for-byte intact. Simultaneously, the magic-link experience is deleted from the v1 critical path because it cannot work without a verified email domain (deferred to Phase 6).

Output: A working code-entry welcome page, a `joinTripByCode` server action + a `/join/[code]` fallback route handler, the magic-link UI/actions/routes removed, and the email-upgrade banner gated off (D-12 depends on Phase 6 email). The `SinCuentaPill` (D-11) stays as the anonymous indicator but no longer triggers an email-upgrade sheet.

Decisions made in this plan:
- `/join/[token]` (uuid) route -> REPLACED by `/join/[code]` (text). Keeping a code-in-URL fallback is low-cost and lets a shared link still work; the old uuid route is deleted to avoid a dead path.
- D-12 upgrade banner -> HIDDEN in v1. Its "Agregar email" CTA depends on Phase 6 email; showing it would dangle a broken flow. The `AnonymousBanner` render is removed from the trip layout. The `AnonymousUpgradeSheet.tsx` and `AnonymousBanner.tsx` FILES are left in the repo unused (Phase 6 reuses them) but are unwired so nothing renders an email-upgrade path.
- D-11 `SinCuentaPill` -> RETAINED as the anonymous-session visual indicator (still valid, no email dependency). Its tap is made inert (no upgrade sheet) in v1.
</objective>

<objective>
Close UAT Gap 2 (Test 5): anonymous join rejects a valid seed invite token.

Root cause (confirmed live, see debug session): RLS chicken-and-egg. `joinTrip()` looks up the
trip by `invite_token` with `.single()` BEFORE the freshly-created anonymous user is a member. The
only trips SELECT policy is `USING (is_trip_member(id))`, so a non-member anon user sees 0 rows →
PGRST116 → mapped to `es.errors.invalidJoinToken`. Service-role returns the row; the identical anon
query returns `[]`. There is no RLS path to resolve a trip by invite_token before membership exists.

Purpose: Restore the anonymous-join entry path (one of the only two ways INTO the app). Unblocks
re-testing UAT Tests 5, 6, 7, 8, 9, 10, 11.

Output: A new migration adding a `SECURITY DEFINER` token-resolution function, and a one-call change
in `joinTrip()` to use it. The strict trips SELECT policy is left UNCHANGED — token resolution is
the only thing that bypasses it, and it returns only the trip id.
</objective>

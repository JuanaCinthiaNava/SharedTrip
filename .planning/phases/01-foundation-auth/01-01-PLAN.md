## Phase Goal

**As a** invited friend, **I want to** open SharedTrip on my iPhone, **so that** I land on a Spanish welcome screen that proves the deployed app is alive, branded, and ready for me to sign in.

> Plan 01 is the Walking Skeleton entry point: greenfield bootstrap → deployed → first user-visible Spanish screen. Subsequent plans (02-05) add the schema, the magic link, the trip shell, and the anonymous-join slice on top of this foundation. **After this plan completes, a user can load `https://{deployed-domain}/` from any device and see the Spanish welcome heading.** That capability did not exist before.

<objective>
Bootstrap the Next.js 16 + TypeScript + Tailwind v4 + shadcn/ui v4 project, create the Supabase cloud project (no schema yet — Plan 02), wire the Tropical Sunset palette + Inter font + `es.ts` dictionary, build the welcome screen as the FIRST end-to-end vertical slice, and deploy to Vercel with all env vars set. This plan establishes every convention every later plan inherits: the `@theme` token names, the `es.ts` lookup pattern, the directory layout, and the deploy pipeline.

Purpose: Prove the deploy pipeline + visual identity + i18n contract are alive before writing any auth code. If this plan fails, no later plan can succeed.

Output: A live Vercel URL serving `/` with the welcome screen in Spanish, on the Tropical Sunset dark palette, with Inter font, and with Supabase + Resend env vars staged in Vercel (not used yet — Plan 03 wires them).
</objective>

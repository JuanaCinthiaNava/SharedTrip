# Phase 2 — UI Design Contract

> Visual and interaction contract for Trip + Member Management. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase EXTENDS a shipped design system — it is NOT greenfield.** Every token, font, color, and component primitive below was locked in Phase 1 (`01-CONTEXT.md` D-05..D-09) and is already live in `src/app/globals.css` + `src/components/`. Phase 2 composes new surfaces from these — it introduces NO new tokens, NO new colors, NO new fonts, and exactly ONE new primitive (`Calendar`, via the official shadcn registry). All decisions trace to `02-CONTEXT.md` D-01..D-18.

## Spacing Scale

Declared values (multiples of 4 only). Inherited from Phase 1 component usage (`gap-4`, `mt-4`, `mt-6`, `py-12`, `py-16` observed in `page.tsx` / `EmptyState.tsx` / `InviteCodeForm.tsx`). Use Tailwind spacing utilities mapped to this scale — never arbitrary px.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon-to-label gaps, badge inner padding |
| sm | 8px | `gap-2` / `p-2` | Compact spacing (avatar↔name, badge gaps) |
| md | 16px | `gap-4` / `p-4` | Default: form field gaps, card padding, member-row vertical rhythm, page horizontal padding (`px-4`) |
| lg | 24px | `gap-6` / `p-6` / `mt-6` | Section padding, CTA top-gap, invite-card internal block spacing |
| xl | 32px | `gap-8` / `py-8` | Layout gaps between distinct sections (invite card ↔ member list) |
| 2xl | 48px | `py-12` | Welcome-screen vertical rhythm, major section breaks |
| 3xl | 64px | `py-16` | Empty-state vertical padding (matches `EmptyState` `py-16`) |

**Exceptions:** Touch targets — every interactive control (buttons, inline "Quitar"/"Salir" actions, calendar day cells, copy button) must be ≥ 44px in the smallest tappable dimension (iOS minimum). Use `min-h-11` (44px) on inline row actions even if their text height is smaller. This is the only allowed deviation from the 4px content scale.

## Typography

LOCKED system rule (Phase 1 D-08): **only weights 400 (regular) and 700 (bold) are permitted across the entire app.** No 500/600. Body line-height 1.5, heading/display line-height 1.2–1.3. Sizes below match the in-code Phase 1 scale (`text-[28px]`, `text-xl`=20px, `text-base`=16px, `text-sm`=14px).

| Role | Size | Weight | Line Height | Usage in Phase 2 |
|------|------|--------|-------------|------------------|
| Display | 28px | 700 | 1.2 (`leading-[1.2]`) | Welcome-screen `h1` (two-choice screen heading) — matches existing `page.tsx` |
| Heading | 20px | 700 | 1.3 (`leading-[1.3]`) | Section headings: "Invita a tu grupo" card title, "Miembros del viaje", create-form title, AlertDialog titles |
| Body | 16px | 400 | 1.5 (`leading-[1.5]`) | Member display names, form inputs, descriptions, dialog body copy, date display, the invite code value |
| Label | 14px | 700 | 1.4 | Field labels ("Nombre del viaje", "Fechas (opcional)"), role badges ("Creador"/"Tú"), the muted helper line in the invite card (use weight 400 for muted helper body, 700 only for the badge/label chrome) |

Notes for executor:
- Role badges render the badge text at 14px/700; secondary/muted helper text (e.g. invite-card body, "Sin fechas todavía") at 14–16px/400 in `text-fg-muted`.
- The invite **code** itself (e.g. `MARR-4F9K`) renders at 20px/700, letter-spaced (`tracking-wide`), in `text-fg` or `text-accent` for emphasis — it is the focal element of the invite card. Use a monospace-feel via `tabular-nums`/`tracking-wide`; do NOT introduce a new mono font.

## Color

60/30/10 split, composed ENTIRELY from the locked Tropical Sunset tokens. NO new colors.

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `#0F1729` midnight navy | `--color-bg` | Page background everywhere |
| Surface (30%) | `#1A2238` | `--color-surface` | Invite card, member rows (subtle surface or transparent on bg), create-form container, AlertDialog/sheet panels, calendar popover surface |
| Accent (10%) | `#FF6B6B` coral (`--color-primary`) + `#3DCCC7` teal (`--color-accent`) | see below | Strictly reserved — see list |
| Destructive | `#EF4444` | `--color-destructive` | Destructive confirm buttons only (remove member, leave trip, delete trip) |
| Mango (notice) | `#FFB627` | `--color-secondary` | Reserved for the static "Sin cuenta" pill (Phase 1, inherited) — NOT introduced as new Phase 2 chrome |
| Muted text | `#94A3B8` | `--color-fg-muted` | Secondary text, labels, "Sin fechas todavía", invite-card helper body, empty-state body |

**Coral (`--color-primary`) reserved for — exhaustive list:**
- Primary CTA buttons: "Quiero crear un viaje", "Crear viaje", "Guardar" (edit), "Copiar invitación"
- The wordmark `SharedTrip` (inherited, in copied-invite preview if shown)
- Active calendar day-range fill / selected-range endpoints
- The `Creador` role badge background (coral-tinted) — see badge spec below

**Teal (`--color-accent`) reserved for — exhaustive list:**
- The invite **code** value emphasis (the literal `MARR-4F9K` string)
- Inline secondary/text-link affordances ("Editar viaje" if rendered as a text affordance rather than a button)
- The deep-link URL fragment if rendered as a visible link
- Calendar "today" indicator ring

**NEVER apply coral or teal to:** all member rows, all body text, all card borders, inactive tabs, the member-list container, or generic hover states. Accent is a focal-point tool, not a default. The bottom tab bar's active-coral treatment is inherited from Phase 1 (D-06) and unchanged.

**Role-badge color spec (D-10 — Claude's discretion, on-brand default chosen):**
- `Creador` badge — coral-tinted pill: `bg-primary/15 text-primary` (low-opacity coral fill, coral text), 14px/700, `rounded-md`, `px-2 py-1`.
- `Tú` badge — neutral/muted pill: `bg-surface text-fg-muted border border-border`, 14px/700, same shape. Keeps coral reserved for the creator's authority signal; "Tú" is informational, not authoritative.
- A user who is BOTH creator and current user shows BOTH badges (`Creador` first, `Tú` second), gap-2.

## New Surface Specs (interaction contracts)

These are the NEW surfaces this phase adds. Behavior is locked in `02-CONTEXT.md`; this section locks the visual/interaction contract.

### 1. Two-choice welcome screen (D-01)
Splits today's code-entry-only `/`. Keep the same `max-w-md px-4 min-h-dvh` shell from `page.tsx`.
- **Join is visually primary** (D-01 / Specifics): "Ya me invitaron" path keeps the existing `InviteCodeForm` as the dominant, sticky-bottom focal element (unchanged from Phase 1).
- "Quiero crear un viaje" is the **secondary** affordance: render as a full-width `Button variant="outline"` (or a clearly-secondary coral-text link) placed below the join form, separated by a divider or a "o" separator label. It must NOT out-weigh the join CTA visually (no solid-coral fill competing with the join submit).
- Tapping "Quiero crear un viaje" → navigates to the create flow (mints anon session per D-02; planner picks route handler vs action).
- Default chosen (auto mode): outline/ghost secondary button labeled `es.trip.createCta`-adjacent welcome label; divider uses `text-fg-muted` "o" centered between two `border-border` rules.

### 2. Create / Edit trip form (D-03, D-14)
Single `CreateTripForm` reused for edit (pre-filled). RHF + Zod (v4), `mode: 'onBlur'` (matches `InviteCodeForm`).
- Fields, in order: **Nombre del viaje** (required, `Input`, 1–80 chars), **Fechas (opcional)** (date-range picker, D-04), **Descripción (opcional)** (multi-line — use a textarea; if a shadcn `Textarea` is not installed, add it via `npx shadcn add textarea`, base-nova).
- Labels 14px/700; inline Zod error under each field via existing `FormMessage` (no toast for validation).
- Submit button full-width coral, `es.trip.saveCta` (create: "Crear viaje"; edit: "Guardar"). Disabled + `Loader2` spinner while pending (matches `InviteCodeForm` pattern).
- Edit surface placement is planner's call (D-14 / RESEARCH Open Q3) — recommended: an "Editar viaje" affordance in the trip header or atop Gente. Whichever placement, it opens this same form pre-filled.

### 3. Date-range picker (D-04)
`TripDatePicker` wrapping the NEW shadcn `Calendar` (`npx shadcn add calendar`, base-nova). Range mode, mobile `numberOfMonths={1}`.
- **Must be clearable** — dates are optional. A visible "Sin fechas todavía" control (text button, `text-fg-muted`) calls `onChange(undefined)`. Do NOT pass `required` to the Calendar (it blocks un-selecting).
- On-brand calendar accents (auto-mode default): selected-range fill = coral (`--color-primary`); range-middle days = `bg-primary/15`; "today" ring = teal (`--color-accent`); month-nav chevrons = lucide, `text-fg`. Calendar surface = `--color-surface`, on `--color-bg`.
- Validate `end >= start` in Zod; error copy `es.trip.invalidDateRange`.
- Display the chosen range inline (below the calendar or in the trigger) via the `es-MX` helper (D-18).

### 4. Member list (Gente tab) (D-10, D-12)
Replaces the `gente/page.tsx` placeholder. RSC fetch of `trip_members` JOIN `profiles`.
- Section heading "Miembros del viaje" (20px/700) — only shown when ≥1 member beyond the invite card; if the user is the sole member, the invite card doubles as the empty state (D-07).
- **MemberRow** layout (horizontal, `gap-4`, `min-h-11`, `py-3`): `UserAvatar` (Phase 1 emoji+color) · display name (16px/400, `text-fg`) · role badge(s) · right-aligned inline action.
- Inline actions (D-12): creator sees "Quitar" on every OTHER row (destructive text affordance, opens AlertDialog); each non-creator sees "Salir del viaje" on their OWN row. The creator has NO "Salir" (D-15 — exit = delete).
- Action affordance style: text button in `text-destructive`, 14px/700, `min-h-11` tap target, right-aligned. Not a solid-fill button (keeps rows calm; the destructive weight lives in the confirm dialog).

### 5. Invite card (D-07, D-08, D-09)
Pinned "Invita a tu grupo" card at the TOP of Gente, above the member list; doubles as the sole-member empty state.
- Card: `--color-surface` background, `rounded-lg`, `p-6`, `gap-4` internal stack.
- Title "Invita a tu grupo" (20px/700) · helper body (14–16px/400, `text-fg-muted`) · the **code** displayed prominently (20px/700, `text-accent`, `tracking-wide`) · single full-width coral `Button` "Copiar invitación" (`es.invite.copyCta`).
- Copy = clipboard ONLY (D-08); on success a Sonner success toast `es.invite.copiedToast` ("Copiado"). On failure (insecure context / blocked) Sonner error toast.
- Copied payload (D-09) carries BOTH the `/join/[code]` deep-link AND the bare code (`es.invite.shareMessage`).

### 6. Confirmation dialogs (D-11, D-17)
All destructive actions use the existing base-ui `AlertDialog`.
- **Remove member:** title `es.members.removeDialogHeading(name)` ("¿Quitar a {nombre} del viaje?"), confirm button destructive-styled `es.members.removeDialogConfirm`, cancel `es.members.cancel`.
- **Leave trip:** title `es.members.leaveDialogHeading(trip)`, destructive confirm `es.members.leaveDialogConfirm`.
- **Delete trip (D-17 — type-to-confirm):** AlertDialog with title `es.trip.deleteDialogHeading`, body `es.trip.deleteDialogBody` (warns of irreversible cascade), an `Input` requiring the user to type the EXACT trip name; the destructive confirm button stays `disabled` until the typed value === trip name (trim-exact match, case-sensitive). Input label `es.trip.deleteConfirmLabel(name)`. This is the highest-stakes action in the app.
- After any mutation: `revalidatePath` + `router.refresh()` (NO realtime in Phase 2 — RESEARCH-resolved). Removed/left member is bounced on next navigation via the layout null-trip guard (D-13) — no dedicated "ya no eres parte" screen.

### 7. es-MX date display (D-18, UI-05)
All trip dates via the shared `formatTripRange` helper (`Intl.DateTimeFormat('es-MX')`, local-parsed — never `.toISOString()`/`new Date(str)`; see RESEARCH Pitfall 2). One helper, inherited by all later phases.
- Range: e.g. "5–12 de jun de 2026"; single: "5 jun 2026"; both null → row hidden entirely (D-03).
- Appears in: trip header, trip switcher list, trip/member cards.

## Loading & Empty & Error States (UI-04 conventions, inherited)

| State | Treatment |
|-------|-----------|
| Loading (member list, trip create) | shadcn `Skeleton` rows (existing primitive); pending buttons show `Loader2` spinner + disabled (matches `InviteCodeForm`) |
| Empty (no co-members) | Invite card IS the empty state (D-07) — no separate empty screen |
| Empty (no trips, switcher) | Inherited Phase 1 copy: "No tienes viajes todavía — pídele un link a quien te invitó o crea uno." + wired "+ Crear nuevo viaje" (D-05) |
| Error (mutation/network) | Sonner toast via `es.errors.*` (e.g. `genericNetwork`); never a raw error string |
| Error (form validation) | Inline `FormMessage` under the field; no toast |
